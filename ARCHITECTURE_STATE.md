## World System Architecture (Phase 2)

```
World "Aethoria" (id=1, seed=42937)
  ├── Monster Types (5): bandit, wolf, ogre, ancient guardian, dragon
  ├── Monster Spawns (28): x/y positions across 10000×10000 coordinate space
  ├── Crystal Nodes (15): 6 types scattered across the world
  └── Kingdoms: each has mapX/mapY assigned on registration
       └── generateKingdomPosition() ensures ≥400 unit separation
```

Terrain is procedurally generated client-side (Unity) and in-engine (lib/game-engine/terrainGenerator.ts) from world seed. No terrain tiles stored in DB.

## Tick System Architecture

```
Server start
  └── runWorldSeed()                    [once on startup]
       ├── ensure Aethoria world (seed=42937)
       ├── create dev@eternalkingdoms.com if absent
       ├── seed 5 monster types if absent
       ├── seed 28 monster spawns if absent
       ├── seed 15 crystal nodes if absent
       └── assign mapX/mapY to unpositioned kingdoms

  └── startResourceTick()               [every 60s]
       ├── load all active kingdoms
       ├── for each kingdom:
       │    ├── applyTick()             [pure, game-engine]
       │    └── UPDATE resources (DB)
       ├── processCompletedUpgrades()
       └── processCompletedConstructions()
```

## Construction Flow (Phase 1.4)

```
POST /kingdoms/:id/construct { buildingType }
  ├── checkCanConstructBuilding()   [slot validation]
  ├── checkAffordability()          [resource validation]
  ├── UPDATE resources (deduct)
  ├── INSERT buildings (level=0, isConstructing=true)
  └── INSERT construction_queue (status=in_progress, endsAt=now+duration)

On next tick (≤60s)
  └── processCompletedConstructions()
       ├── UPDATE buildings SET level=1, isConstructing=false
       ├── UPDATE construction_queue SET status=completed
       └── computeKingdomPower() → UPDATE kingdoms.power
```
